Configure Tasks - Placement Options

These mockups show different ways to integrate the Configure Tasks interface into the Onboarding Wizard. Currently it opens as a separate panel. Here are alternatives that make it feel more seamless.

Option A: Current Implementation (Slide-out Panel)

Current behavior: User clicks "Configure Tasks" button, which opens a large slide-out panel from the right. The wizard remains visible but dimmed in the background.

Pros

  • Already implemented
  • Fluent UI standard pattern
  • Can be dismissed easily

Cons

  • Feels disconnected from wizard flow
  • Context switch - user leaves the wizard mentally
  • Inconsistent with other wizard steps

Option B: Full Overlay (Exact Same Dimensions) Recommended

Configure Tasks opens as an overlay that exactly matches the wizard dimensions. It appears to "replace" the wizard content while maintaining the same footprint. The sidebar shows a special "Configure Tasks" state with a back button.

Pros

  • Same visual footprint - feels integrated
  • Clear back navigation to return
  • Maintains context within wizard flow
  • More screen real estate for task config

Cons

  • Slightly more complex implementation
  • User can't see wizard state while configuring

Option C: Inline Expansion (Accordion Style)

The task list and configuration expand inline within the wizard step content area. No overlays or panels - everything stays in place. Works well for simpler configurations.

Pros

  • No context switch at all
  • Sidebar and tips remain visible
  • Feels like a natural wizard step

Cons

  • Limited horizontal space for split view
  • May feel cramped with many tasks
  • Scrolling can be confusing

Option D: Replace Wizard Content Directly

When user clicks Configure Tasks, the wizard content area is directly replaced with the task configuration UI. The wizard sidebar remains but shows a modified state. Simplest "in-place" solution.

Pros

  • Simplest implementation
  • No overlay or panel management
  • Wizard sidebar provides navigation context

Cons

  • Tips panel may not be relevant
  • Less visually distinct from wizard steps

Recommended: Option B (Full Overlay with Same Dimensions)

This option provides the best balance of:

  • Visual consistency with the wizard (same footprint)
  • Clear navigation with "Back to Wizard" button
  • Full screen real estate for task configuration
  • Task codes prominently displayed (DOC-001, SYS-001, etc.)
  • Summary sidebar showing category breakdown and progress
